# Play Store Telegram Bot 🤖

Бот стежить за додатками в Google Play за **package name (bundle ID)** і надсилає
тобі повідомлення в Telegram:
- 🚀 коли додаток **вийшов** у стор (став доступний);
- ⛔ коли додаток **зняли / забанили** (став недоступний).

Працює **безкоштовно** у хмарі через **GitHub Actions** — без сервера, без оплат, без картки.

---

## Як це працює

Бот працює **самопідтримним ланцюжком** на GitHub Actions (бо вбудований розклад GitHub
ненадійний). Кожен запуск `check.py`:
1. на початку тригерить наступний запуск (через секрет `WORKFLOW_PAT`) — тому ланцюжок
   не переривається навіть якщо щось впаде;
2. перевіряє статус кожного додатку на Google Play і шле сповіщення при зміні;
3. ~4.5 хвилини активно «слухає» повідомлення (long polling) → відповіді майже миттєві;
4. зберігає стан у `state.json`.

Розклад `*/5` лишено як запасний «будильник» — якщо ланцюжок колись обірветься, він його оживить.

## Налаштування (10 хвилин, робиться один раз)

### Крок 1. Створити Telegram-бота
1. У Telegram відкрий **@BotFather**.
2. Надішли `/newbot`, дай ім'я та username.
3. BotFather дасть тобі **токен** виду `123456789:AAG...`. Скопіюй його.

### Крок 2. Створити репозиторій на GitHub
1. Зайди на https://github.com → **New repository**.
2. Назви як хочеш, наприклад `playstore-bot`. Можна **Private** (приватний).
3. Створи порожній репозиторій.

### Крок 3. Залити ці файли в репозиторій
Найпростіше — через термінал (заміни `USERNAME/REPO` на свій):

```bash
cd playstore-telegram-bot
git init
git add .
git commit -m "init play store bot"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
```

Або просто перетягни всі файли через кнопку **"Add file → Upload files"** на сайті GitHub.

### Крок 4. Додати токен як секрет
1. У репозиторії: **Settings → Secrets and variables → Actions → New repository secret**.
2. Name: `BOT_TOKEN`
3. Secret: встав токен від BotFather.
4. **Add secret**.

### Крок 5. Увімкнути Actions
1. Вкладка **Actions** у репозиторії → якщо просить, натисни **"I understand my workflows, enable them"**.
2. Обери workflow **"Play Store Monitor"** → **Run workflow** (щоб перевірити одразу, не чекаючи 5 хв).

Готово! ✅

## Як користуватись

1. Відкрий свого бота в Telegram і натисни **Start**.
2. Надішли йому package name додатку, напр.:
   ```
   com.instagram.android
   ```
3. Бот відповість, чи додаток вже в сторі, і буде стежити далі.
4. Коли статус зміниться — прийде сповіщення.

**Команди:**
- `/list` — показати, за чим стежу
- `/remove com.example.app` — прибрати додаток
- `/help` — довідка

> 📦 Package name (bundle ID) — це не назва додатку, а його технічний ID виду `com.company.app`.
> Його видно в URL сторінки додатку в Google Play: `.../details?id=**com.company.app**`.

## Важливі нюанси (щоб працювало завжди)

- **Секрет `WORKFLOW_PAT` живить самозапуск.** Це GitHub-токен (scope `repo`), яким кожен
  запуск тригерить наступний. ⚠️ Якщо видалити цей токен у GitHub — ланцюжок зупиниться.
  Тоді створи новий токен і онови секрет **Settings → Secrets → Actions → WORKFLOW_PAT**.
- **Тільки один бот має читати повідомлення.** Не запускай `check.py` локально одночасно
  з GitHub Actions — інакше повідомлення "з'їсться" одним із них.
- **Ліміти безкоштовно:** для **публічного** репозиторію хвилини Actions безлімітні — саме тому
  тут безперервний ланцюжок безкоштовний. На приватному репо (2000 хв/міс) така схема ліміт
  вичерпає — тоді або зроби репо Public, або відмовся від самозапуску й покладайся на розклад.
- **Якщо ланцюжок колись зупиниться** (напр. GitHub поскаржиться на безперервні запуски) —
  просто зайди в **Actions → Play Store Monitor → Run workflow**, і він знову піде по колу.
